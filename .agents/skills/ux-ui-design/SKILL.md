# UX/UI Design Skill

## Goal

Create an accessible, credible, and efficient conference discovery interface for researchers, PhD students, and academic groups.

## Design principles

- Prioritize fast discovery of deadlines.
- Make uncertainty visible.
- Keep source provenance visible.
- Avoid visual clutter.
- Design for mobile and desktop.
- Use semantic HTML and accessible components.
- Do not hide critical information behind hover-only interactions.

## Core pages

Design and implement:

- homepage;
- conference listing page;
- conference detail page;
- deadline calendar or timeline page;
- optional admin/review queue if confirmed;
- about/methodology page explaining source policy and update frequency.

## Core components

Create components for:

- search bar;
- filter panel;
- sort selector;
- conference card;
- deadline badge;
- fee summary;
- source/provenance block;
- confidence/review status label;
- country/location display;
- theme tags;
- loading state;
- empty state;
- error state.

## Filter requirements

Support filters for:

- field;
- country;
- date range;
- submission deadline range;
- fee availability;
- mode: in-person, hybrid, online, unknown;
- theme/topic;
- review/confidence status if appropriate.

## Accessibility expectations

Check:

- keyboard navigation;
- focus states;
- form labels;
- color contrast;
- ARIA only where needed;
- responsive layout;
- readable date formatting;
- reduced ambiguity in badges and icons.

## Design confirmation

Before implementing a full visual system, ask the user to choose between concise options, for example:

1. Minimal academic directory.
2. Modern research dashboard.
3. Sci-tech editorial style.
4. Dense database/search-engine style.

Recommend one option, but wait for confirmation.

## Output format

Return:

- proposed information architecture;
- page/component plan;
- design choices needing confirmation;
- implemented files;
- accessibility notes;
- screenshots or local preview notes when available.
